# Refactoring Towards Microservices: Preparing the Ground for Service Extraction

## Summary

This arXiv paper introduces a catalogue of seven refactorings aimed at preparing monolithic codebases for microservices extraction, with emphasis on managing dependencies at the code level. It argues that existing decomposition guidance is mostly architectural and under-specified for day-to-day developer work. The catalogue consolidates literature-backed refactorings into a structured, step-by-step process intended to simplify migration and support future automation.

## Key claims

- Monolith-to-microservices migration remains largely manual and labour-intensive despite growing organizational demand for scalability, availability, and independent deployments.
- Prior work on monolith decomposition focuses mainly on architecture-level strategies and does not adequately address code-level challenges and dependencies.
- The authors propose a systematic catalogue of seven refactorings specifically designed to support dependency handling during service extraction.
- The catalogue synthesizes refactorings identified in prior literature while filling a gap in code-level systematization of the migration process.
- A structured, step-by-step refactoring approach can reduce migration complexity and create a foundation for automating parts of the transition.
