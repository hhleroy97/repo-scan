# 2026-06-09 — Phase A: robust initial digest

Build-order phase A complete. `repo-scan` v0.2.0.

## What changed

- **Test harness (A.0)** — `tests/` with pytest: tmp-repo fixtures, 23 tests locking
  scan outputs, config loading, hook idempotency, and all Phase A features.
- **madge fix (A.1)** — TS/JS dep graphs now use `madge --json` and build Mermaid in
  Python (madge 8 removed `--mermaid`; the old call could never succeed). Skip
  messages now distinguish "tool missing" from "no files of that language".
- **Identity block (A.2)** — `index.md` opens with README summary, remote, manifests
  (`detect_manifests`), and entry points (`detect_entry_points`: pyproject scripts,
  package.json bin/main, convention files).
- **Directory tree (A.3)** — depth-capped ASCII tree (`tree_depth`, default 3) in
  `index.md`, honoring `exclude_dirs`.
- **Important-files ranking (A.4)** — "Start here" table: composite score of import
  centrality (intra-repo dep edges) x git churn x complexity x size. Generated docs
  excluded so the scan never ranks its own output.
- **Machine sidecar (A.5)** — `docs/scan.json` (schema_version 1): identity, files,
  churn, complexity, ranking, dependency edges, config. Agents read this instead of
  parsing markdown.
- **AGENTS.md scaffold (A.6)** — `repo-scan --init-agents` writes ownership/gate/
  off-limits rules from the RADAR context template. Never overwrites.
- **RADAR candidates (A.7)** — with `"radar_enabled": true`, writes
  `docs/research/candidates.md`: high-churn x high-complexity files, the future loop
  trigger feed.
- **Digest export (A.8)** — `repo-scan --digest` writes a token-budgeted
  (`digest_tokens`) single-file `docs/digest.md` for pasting into any LLM.
- `--version` flag added (HANDOFF noted it missing); config loader warns on unknown
  keys; version bumped to 0.2.0.

## Verification

- Scanned this repo and a fixture repo; `scan.json` ranking and edges spot-checked.
- `--digest`, `--init-agents`, `--version` exercised manually, exit 0.
