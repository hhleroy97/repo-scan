# 2026-06-10 — The Act stage: approved spec to reviewed commit

> [!success] RADAR now closes the loop end-to-end
> Research -> Analyze (gate) -> Draft -> Audit (gate) -> **Act** (two gates) -> Record.
> A ticket approved from the phone can now end as a tested commit on a review
> branch, with every decision point still human-gated.

## What was built

- `repo_scan/radar/act.py` — the Act stage (`radar act`):
  - picks the highest-priority in-progress ticket with an approved spec
    (wikilink parsed from ticket notes, spec must carry `status: "approved"`)
  - **Gate `pre_implement`** before any work
  - isolated branch `radar/<ticket-id>`; refuses dirty trees (vault churn
    under `docs/` is exempt — the agent is forbidden from touching it)
  - agent CLI implements the spec with repo cwd (`act_timeout`, default 1800s)
  - **tests as the hard gate**: `test_cmd` (auto-detected if unset) must
    pass; failures are fed back to the agent for bounded fix rounds
    (`act_fix_rounds`, default 2); persistent failure leaves the branch for
    human review and notes the ticket
  - **Gate `post_implement`** with diff stat before anything is committed
  - commit excludes vault churn; ticket gets an `implemented:` note pointing
    at branch + commit; outcome recorded to `changelog/{date}-act.md`
  - every stage checkpoints, so daemon resumes skip the expensive agent call

- Daemon integration: act candidates are picked up automatically (before new
  research loops — finish started work first), runs carry `kind: act`, gate
  pauses notify via ntfy with dashboard deep links. The phone flow is now:
  approve ticket -> research loop gates -> spec approved -> pre_implement ->
  post_implement (diff stat in the gate detail) -> "implementation committed"
  notification.

- `gates.py`: gate registry extended with `pre_implement` / `post_implement`
  (default `prompt` — autonomy is opt-in per gate, as everywhere else).
- `llm.complete` learned `cwd` so act-mode agents edit the right repo.
- Config keys: `act_enabled` (off by default), `test_cmd`, `test_timeout`,
  `act_timeout`, `act_fix_rounds`.

## Also

- Hub auto-start on boot: a hidden VBS launcher in the Windows Startup folder
  starts `radar serve` inside WSL at logon (guarded — no-op if already
  running). Tailscale serve config persists on its own.

## Tests

8 new offline tests in `tests/test_act.py` using a stub "agent" script that
actually edits the repo: happy path (branch, commit, vault exclusion, ticket
note, checkpoint cleanup), dirty-tree refusal, pause/resume through both
gates via the decision inbox (verifying the agent is not re-invoked on
resume), post-implement rejection, fix-round recovery, fix-round exhaustion,
and daemon act-run scheduling. 148 passing total.
